## Переменные окружения

Все переменные окружения управляются через модуль `src/config.lib.mjs`, который использует `getenv` для надёжной обработки. Конфигурация использует имена свойств в формате camelCase в соответствии с соглашениями JavaScript.

### 5.1. Настройки TTL кэша

Эти параметры определяют, как долго кэшируются ответы API перед выполнением нового запроса.

| Переменная окружения               | По умолчанию | Описание                                                                                                                                                              |
| ---------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `HIVE_MIND_API_CACHE_TTL_MS`       | 180000       | Общий TTL кэша API в мс (3 минуты). Используется для GitHub API.                                                                                                      |
| `HIVE_MIND_USAGE_API_CACHE_TTL_MS` | 600000       | TTL кэша Claude Usage API в мс (10 минут). **Важно:** Claude Usage API имеет более строгое ограничение частоты запросов. Слишком частые вызовы могут возвращать null. |
| `HIVE_MIND_SYSTEM_CACHE_TTL_MS`    | 120000       | TTL кэша системных метрик в мс (2 минуты). Используется для RAM, CPU и дискового пространства.                                                                        |

**Примечание:** Claude Usage API (`/api/oauth/usage`) имеет более строгое ограничение частоты запросов по сравнению с другими API. Если вы видите `null`-значения в выводе команды `/limits`, возможно, частота вызовов API слишком высока. Значение TTL по умолчанию (10 минут) разработано для предотвращения этой проблемы. Подробности см. в [Issue #1074](https://github.com/link-assistant/hive-mind/issues/1074).

### 5.2. Настройки Claude Code CLI

Эти параметры управляют поведением Claude Code CLI, включая лимиты вывода и таймауты MCP.

| Переменная окружения                    | По умолчанию | Описание                                                                                                        |
| --------------------------------------- | ------------ | --------------------------------------------------------------------------------------------------------------- |
| `CLAUDE_CODE_MAX_OUTPUT_TOKENS`         | 64000        | Максимальное количество токенов вывода для Claude Code CLI (также: `HIVE_MIND_CLAUDE_CODE_MAX_OUTPUT_TOKENS`)   |
| `CLAUDE_CODE_MAX_OUTPUT_TOKENS_OPUS_46` | 128000       | Максимальное количество токенов вывода для Opus 4.6+ (также: `HIVE_MIND_CLAUDE_CODE_MAX_OUTPUT_TOKENS_OPUS_46`) |
| `MCP_TIMEOUT`                           | 900000       | Таймаут запуска MCP-сервера в мс (15 мин) (также: `HIVE_MIND_MCP_TIMEOUT`)                                      |
| `MCP_TOOL_TIMEOUT`                      | 900000       | Таймаут выполнения инструмента MCP в мс (15 мин) (также: `HIVE_MIND_MCP_TOOL_TIMEOUT`)                          |
| `HIVE_MIND_MAX_THINKING_BUDGET_OPUS_46` | 31999        | Максимальный бюджет токенов для размышлений для моделей Opus 4.6+                                               |

**Примечание:** Модели Claude поддерживают разное максимальное количество токенов вывода: Opus 4.6 (псевдоним `opus` по умолчанию) поддерживает 128K токенов, тогда как Sonnet 4.5, Opus 4.5 и Haiku 4.5 поддерживают 64K токенов. Таймауты MCP (по умолчанию 15 минут) учитывают длительные операции Playwright. Подробности см. в [Issue #1076](https://github.com/link-assistant/hive-mind/issues/1076) и [Issue #1066](https://github.com/link-assistant/hive-mind/issues/1066).

### 17. Playwright MCP

Playwright MCP (Model Context Protocol) предоставляет возможности автоматизации браузера для Claude Code, обеспечивая веб-скрапинг, UI-тестирование и взаимодействие с динамическими веб-страницами.

#### Установка

```bash
# Рекомендуется: Установка с настройками безопасности памяти (для серверов и Docker)
claude mcp add playwright -s user -- npx -y @playwright/mcp@latest --isolated --headless --no-sandbox --timeout-action=600000 --viewport-size 1920x1080

# Минимальная установка (для локальной разработки)
claude mcp add playwright -s user -- npx -y @playwright/mcp@latest --isolated --headless
```

#### Аргументы командной строки

| Аргумент                 | Описание                                               | Влияние на память                                       |
| ------------------------ | ------------------------------------------------------ | ------------------------------------------------------- |
| `--isolated`             | Эфемерные контексты браузера (НАИБОЛЕЕ ВАЖНО)          | **ВЫСОКОЕ** — предотвращает накопление процессов        |
| `--headless`             | Запуск браузера в безголовом режиме                    | **СРЕДНЕЕ** — снижает накладные расходы UI на память    |
| `--browser <type>`       | Браузер: chromium, firefox, webkit, msedge             | **ВАРЬИРУЕТСЯ** — WebKit часто использует меньше памяти |
| `--no-sandbox`           | Отключить sandbox (только для контролируемых сред)     | **НИЗКОЕ** — незначительно снижает память               |
| `--timeout-action <ms>`  | Таймаут для действий (по умолчанию: 5000)              | **Н/П** — предотвращает зависшие процессы               |
| `--viewport-size <size>` | Установить размеры viewport (например, "1280x720")     | **НИЗКОЕ** — влияет на память рендеринга                |
| `--storage-state <path>` | Загрузить состояние аутентификации без полного профиля | **СРЕДНЕЕ** — аутентификация без раздувания профиля     |

#### Параметры области видимости

| Область   | Описание                         | Расположение конфигурации                 |
| --------- | -------------------------------- | ----------------------------------------- |
| `local`   | Только текущая директория        | `~/.claude.json` (специфично для проекта) |
| `project` | Совместная для команды через VCS | `.mcp.json` (корень проекта)              |
| `user`    | Доступна глобально               | `~/.claude.json` (раздел пользователя)    |

#### Конфигурация JSON

Прямая конфигурация в `~/.claude.json`:

```json
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest", "--isolated", "--headless", "--no-sandbox", "--timeout-action=600000", "--viewport-size", "1920x1080"],
      "env": {
        "PLAYWRIGHT_BROWSERS_PATH": "/opt/playwright/browsers"
      }
    }
  }
}
```

#### Команды MCP

```bash
# Список настроенных MCP-серверов
claude mcp list

# Получить информацию о сервере
claude mcp get playwright

# Удалить сервер
claude mcp remove playwright
```

#### Рекомендации

1. **Всегда используйте режим `--isolated`** — предотвращает накопление процессов Chrome и утечки памяти
2. **Закрепите конкретную версию** — используйте `@playwright/mcp@0.0.49` вместо `@latest` для стабильности
3. **Используйте `--headless` для серверов** — снижает накладные расходы на память в CI/CD и продакшн-средах
4. **Периодически перезапускайте Claude Code** — для долгих сессий для освобождения накопленных ресурсов браузера

Подробные параметры конфигурации, устранение неполадок и расширенные сценарии использования см. в подробном руководстве:
[Руководство по настройке Playwright MCP](./case-studies/issue-837-playwright-mcp-chrome-leak/04-CLAUDE-PLAYWRIGHT-MCP-CONFIGURATION.md)

## Примеры использования

### Установка переменных окружения

```bash
# Увеличить таймаут Claude до 2 минут
export HIVE_MIND_CLAUDE_TIMEOUT_SECONDS=120

# Уменьшить задержку GitHub API для более быстрых операций
export HIVE_MIND_GITHUB_API_DELAY_MS=2000

# Увеличить порог автопродолжения до 48 часов
export HIVE_MIND_AUTO_CONTINUE_AGE_HOURS=48

# Использовать кастомную временную директорию
export HIVE_MIND_TEMP_DIR=/var/tmp/hive-mind

# Включить отслеживание ошибок Sentry (по умолчанию отключено)
export HIVE_MIND_SENTRY=true

# Настроить для GitHub Enterprise
export HIVE_MIND_GITHUB_BASE_URL=https://github.enterprise.com
```

### Запуск с пользовательской конфигурацией

```bash
# Запуск с кастомными таймаутами
HIVE_MIND_CLAUDE_TIMEOUT_SECONDS=120 HIVE_MIND_RETRY_BASE_DELAY_MS=10000 hive https://github.com/owner/repo

# Запуск с увеличенными лимитами
HIVE_MIND_GITHUB_FILE_MAX_SIZE=52428800 HIVE_MIND_MIN_DISK_SPACE_MB=1000 solve https://github.com/owner/repo/issues/123

# Запуск с кастомными настройками автопродолжения (--auto-continue включён по умолчанию)
HIVE_MIND_AUTO_CONTINUE_AGE_HOURS=12 solve https://github.com/owner/repo/issues/456
```

### Файл конфигурации (необязательно)

Вы можете создать файл `.env` в корне проекта:

```bash
# файл .env
HIVE_MIND_CLAUDE_TIMEOUT_SECONDS=90
HIVE_MIND_GITHUB_API_DELAY_MS=3000
HIVE_MIND_AUTO_CONTINUE_AGE_HOURS=36
HIVE_MIND_TEMP_DIR=/opt/hive-mind/tmp
HIVE_MIND_SENTRY_DSN=your-custom-sentry-dsn
```

Затем загрузите его перед запуском:

```bash
source .env
hive https://github.com/owner/repo
```

import { timeouts, githubLimits, sentry } from './config.lib.mjs';

- Все ограничения размера указаны в байтах, если не указано иное
