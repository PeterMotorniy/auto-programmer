## Summary

This PR adds an experimental `/merge` command to the hive-telegram-bot that allows processing a repository's merge queue - merging all PRs with the 'ready' label sequentially, one by one.

### Features

- **`/merge <repository-url>`** - Start merging all ready PRs
  - Accepts GitHub repository URLs (e.g., `https://github.com/owner/repo`)
  - Automatically creates 'ready' label if it doesn't exist (requires write permission)
  - Finds PRs with 'ready' label directly or via linked issues
  - Sorts PRs by creation date (oldest first, using minimum of PR and linked issue dates)
  - Merges sequentially, waiting for CI/CD to pass after each merge (checks every 5 minutes)
  - Provides real-time progress updates via single updated Telegram message
  - Cancel via inline button (no separate commands)

### Architecture

The implementation follows the existing modular pattern in the codebase:

```
src/config.lib.mjs                - Merge queue ENV configuration
src/github-merge.lib.mjs          - GitHub label and merge operations
src/telegram-merge-queue.lib.mjs  - Sequential merge processor class
src/telegram-merge-command.lib.mjs - Telegram command handler
tests/test-merge-queue.mjs        - Unit tests (24 tests)
```

### User Experience

- Progress displayed with emojis (⏳ pending, ✅ merged, ❌ failed, ⏭️ skipped)
- Progress bar rendered in code block for better style (like /limits command)
- Single updated message (no separate status command)
- Cancel button in message (no separate cancel command)
- User-friendly error messages (debug info only in --verbose mode)

### Configuration (ENV variables)

| Variable | Default | Description |
|----------|---------|-------------|
| `HIVE_MIND_MERGE_QUEUE_MAX_PRS` | 10 | Maximum PRs per session |
| `HIVE_MIND_MERGE_QUEUE_CI_POLL_INTERVAL_MS` | 300000 (5 min) | CI check interval |
| `HIVE_MIND_MERGE_QUEUE_CI_TIMEOUT_MS` | 25200000 (7 hours) | CI timeout |
| `HIVE_MIND_MERGE_QUEUE_POST_MERGE_WAIT_MS` | 60000 (1 min) | Wait after merge |

### Safety Features

- Only one merge operation per repository at a time (not per chat)
- CI/CD status monitoring with configurable timeout (7 hours default)
- Graceful cancellation (completes current PR before stopping)
- Skips unmergeable PRs (conflicts, draft, behind base branch)
