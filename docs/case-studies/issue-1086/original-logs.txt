## Observations

1. **Solve overrides shows 5 options**:
   - --attach-logs
   - --verbose
   - --no-tool-check
   - --auto-resume-on-limit-reset
   - --tokens-budget-stats

2. **Hive overrides shows only 6 options** (2 missing):
   - --all-issues
   - --once
   - --skip-issues-with-prs
   - --attach-logs
   - --verbose
   - --no-tool-check

   **Missing options:**
   - --auto-resume-on-limit-reset
   - --tokens-budget-stats

3. **Both validations pass**: The issue is not detected during validation because
   yargs validation receives all options (the nested structure is still parsed
   as valid options), but the display logic in `lino.format()` only shows
   top-level values.

## Key Finding

The difference between TELEGRAM_SOLVE_OVERRIDES and TELEGRAM_HIVE_OVERRIDES:
- SOLVE: Each option is on a separate line (works correctly)
- HIVE: Last two options are on the same line (causes the bug)
