## Follow-up after PR review

[PR comment #4678094849](https://github.com/link-assistant/hive-mind/pull/1892#issuecomment-4678094849)
asked to **split each queue into separate lists (pending, processing, completed)**
rather than merging executing + pending into one bullet list. The detailed status
now renders, per tool, four individually-labeled lists — `*Processing*`,
`*Pending*`, `*Completed*`, `*Failed*` — each with its own count and nested
items. Completed/Failed history is grouped per tool (`groupQueueItemsByTool`) and
a tool with only history (drained live queue) still shows its history lists.

## Explicitly out of scope (and why)

- **`formatStatus`** (the one-line-per-tool summary used by `/limits`) was left
  unchanged: it never listed individual items, so it had no duplication to remove.
  Touching it would change an unrelated surface.
- The **Completed / Failed history sections** keep their `MAX_DISPLAY_ITEMS_PER_QUEUE`
  cap with the "… and N more" collapse — an unbounded history list would be noise,
  not data.
