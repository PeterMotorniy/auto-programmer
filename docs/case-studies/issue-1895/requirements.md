## Notes on R-11 / R-15 (external reporting & the linking-API question)

Two separate targets, two decisions:

- **`link-foundation/meta-language`** (PRs #65/#66, issues #49/#50): **no source
  bug** to report — the PR bodies had the correct `Fixes #N` keywords; the defect
  was the **hive-mind workflow** creating PRs against a non-default branch and not
  compensating. Fixed here.
- **GitHub platform** (raised by the maintainer on #1896): we confirmed by **live
  GraphQL schema introspection** that there is **no public API** to link an
  existing PR to an issue (the Development-sidebar action), no non-default-branch
  auto-close, and no API to even _read_ a PR's linked issues. We **did not miss an
  API — it does not exist.** We reported the gap upstream: upvoted the three
  canonical feature requests (#112224, #155339, #179613) and posted a reproducible
  evidence comment on #155339. Full method, proof and reproduction in
  [`github-api-linking-research.md`](./github-api-linking-research.md); decision
  rationale in [`external-report.md`](./external-report.md).

The earlier note here said "no GitHub-platform bug … to report." That was too
narrow: the auto-close _rule_ is documented (correct), but the **missing linking
API** is a real, reportable gap — now reported.
