    fix: increase Claude Usage API cache TTL to 20 minutes to avoid rate limiting
    
    The Claude Usage API (/api/oauth/usage) was returning null values for all
    usage metrics because the 3-minute cache TTL was too aggressive for
    Anthropic's rate limiting policy.
    
    Changes:
    - Add configurable cache TTL for Usage API via HIVE_MIND_USAGE_API_CACHE_TTL_MS
    - Default to 20 minutes (1200000ms) for Claude Usage API
    - Keep 3 minutes for regular APIs (GitHub) and 2 minutes for system metrics
    - Add HTTP response status logging for debugging
    - Add explicit 429 rate limit error handling
    - Add case study documentation for issue #1074
    - Add test script for cache TTL validation
    
    Configuration:
    - HIVE_MIND_API_CACHE_TTL_MS: General API cache (default: 180000ms = 3 min)
    - HIVE_MIND_USAGE_API_CACHE_TTL_MS: Claude Usage API (default: 1200000ms = 20 min)
    - HIVE_MIND_SYSTEM_CACHE_TTL_MS: System metrics (default: 120000ms = 2 min)
    
    Fixes #1074
