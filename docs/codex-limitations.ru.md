# Ограничения инструмента Codex (languages: [en](codex-limitations.md) • [zh](codex-limitations.zh.md) • [hi](codex-limitations.hi.md) • ru)

## Сетевые ограничения

Инструмент Codex работает в изолированной среде с ограниченным доступом к сети из соображений безопасности.

### Что это означает

1. **Невозможность push в GitHub**: Codex не может напрямую выполнить `git push`
2. **Невозможность получения внешних ресурсов**: Ограниченный доступ к внешним API
3. **Невозможность выполнения сетевых команд**: Команды вроде `curl`, `wget` могут завершаться с ошибкой

### Как solve.mjs справляется с этими ограничениями

Скрипт solve обходит эти ограничения следующим образом:

1. **Начальная настройка**: solve.mjs клонирует репозиторий и создаёт ветку (до запуска codex)
2. **Выполнение Codex**: Codex создаёт и изменяет файлы, делает коммиты локально
3. **Автоматический перезапуск**: Если codex оставил незакоммиченные изменения, solve.mjs автоматически перезапускает codex
4. **Финальный push**: После завершения codex, solve.mjs отправляет изменения в GitHub (вне изолированной среды)

### Ожидаемый рабочий процесс

```
[solve.mjs] Clone repo and create branch
            ↓
[codex]     Make changes and commit locally
            ↓
[solve.mjs] Detect uncommitted changes? → Restart codex
            ↓
[codex]     Commit remaining changes
            ↓
[solve.mjs] Push all commits to GitHub
            ↓
[solve.mjs] Exit successfully
```

### Устранение неполадок

Если в выводе codex вы видите «Could not resolve host: github.com»:

- ✅ Это ожидаемо и нормально
- ✅ solve.mjs выполнит push после завершения codex
- ⚠️ Не прерывайте работу нажатием Ctrl+C — дождитесь завершения процесса

Если solve.mjs не выполняет push после завершения codex:

- Проверьте, не прерывали ли вы процесс раньше времени
- Выполните push вручную: `git push origin <branch-name>`
- Сообщите об ошибке, если проблема повторяется постоянно

## Автоматический перезапуск vs. Режим наблюдения

### Автоматический перезапуск (временный мониторинг)

Когда codex или другие инструменты оставляют незакоммиченные изменения, solve.mjs автоматически переходит в «Режим автоматического перезапуска»:

- **Назначение**: Завершить незаконченную работу из предыдущего запуска
- **Триггер**: Обнаруженные незакоммиченные изменения после выполнения инструмента
- **Продолжительность**: Запускается один раз, завершается после коммита изменений
- **НЕ то же самое, что**: Запрошенный пользователем режим `--watch`

**Пример вывода:**

```
🔄 AUTO-RESTART: Uncommitted changes detected
   Starting temporary monitoring cycle (NOT --watch mode)
   The tool will run once more to commit the changes
   Will exit automatically after changes are committed

🔄 AUTO-RESTART MODE ACTIVE
   Purpose: Complete unfinished work from previous run
   Monitoring PR: #123
   Mode: Temporary (NOT user-requested --watch)
```

### Режим наблюдения (непрерывный мониторинг)

При явном использовании `--watch`, solve.mjs непрерывно отслеживает обратную связь:

- **Назначение**: Непрерывный мониторинг обратной связи пользователя по PR
- **Триггер**: Пользователь указывает флаг `--watch`
- **Продолжительность**: Работает бесконечно до слияния PR или нажатия Ctrl+C
- **Сценарий использования**: Долгосрочный мониторинг обратной связи

**Пример вывода:**

```
👁️ WATCH MODE ACTIVATED
   Checking interval: 60 seconds
   Monitoring PR: #123
   Stop condition: PR merged by maintainer
```

### Ключевые различия

| Функция           | Автоматический перезапуск                  | Режим наблюдения          |
| ----------------- | ------------------------------------------ | ------------------------- |
| Активация         | Автоматическая (незакоммиченные изменения) | Ручная (флаг `--watch`)   |
| Продолжительность | Один цикл                                  | Непрерывная               |
| Назначение        | Завершить незаконченную работу             | Мониторинг обратной связи |
| Условие выхода    | Изменения закоммичены                      | PR слит или Ctrl+C        |

## Распространённые проблемы

### Проблема: «Режим наблюдения активирован, но я не использовал --watch»

**Пояснение**: Это режим автоматического перезапуска, а не запрошенный пользователем режим наблюдения.

**Причина**: Инструмент оставил незакоммиченные изменения, которые необходимо обработать.

**Решение**: Дождитесь завершения процесса. Он автоматически завершится после коммита изменений.

### Проблема: «Codex не может выполнить push в GitHub»

**Пояснение**: Codex работает в изолированной среде без доступа к сети.

**Причина**: Ограничения безопасности в среде выполнения Codex.

**Решение**: solve.mjs автоматически отправит изменения после завершения Codex. Не прерывайте процесс.

### Проблема: «Процесс, кажется, завис в режиме наблюдения»

**Пояснение**: Либо автоматический перезапуск ожидает коммита изменений, либо вы использовали `--watch`.

**Диагностика**:

1. Проверьте сообщения журнала — там написано «AUTO-RESTART MODE» или «WATCH MODE ACTIVATED»?
2. Если автоматический перезапуск: проверьте, есть ли ещё незакоммиченные изменения
3. Если режим наблюдения: вы использовали флаг `--watch`, дождитесь слияния PR или нажмите Ctrl+C

**Решение**:

- Для автоматического перезапуска: дождитесь завершения или закоммитьте изменения вручную
- Для режима наблюдения: дождитесь завершения или прервите нажатием Ctrl+C
